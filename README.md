# Sales & Reporting Automation

## 📌 Project Overview
This project simulates a real Business Analyst assignment for a D2C e-commerce company in the health supplements domain.

The objective was to investigate a business problem:
Revenue increased by 18%, but customer complaints increased by 35%.

The project includes data analysis, SQL queries, dashboard creation, and automation of weekly reporting.

---

## 📊 Dataset
The dataset consists of 5 tables:
- Customers
- Orders
- Order Items
- Products
- Complaints

---

## 🔍 Key Analysis & Insights

### 1. Complaint Analysis
- Calculated complaint rate instead of absolute complaints
- Identified courier partner with higher complaint rate
- Analysed complaint types (delivery, product issues)

### 2. Customer Analysis
- Total orders and spend per customer
- Identification of high-value customers
- Complaint count per customer

### 3. Product Analysis
- Frequently bought product pairs
- Category-level revenue trends

---

## 📈 Dashboard (Excel / Google Sheets)
- KPIs: Revenue, Orders, AOV, Delivery %
- Dynamic city-level filtering
- Month-on-Month growth tracking
- Conditional formatting for insights

---

## ⚙️ Automation (Key Highlight)
Automated weekly reporting process:
- Consolidated data using SQL
- Connected Excel using Power Query
- Built one-click refresh dashboard
- Reduced manual effort from 2 hours to <15 minutes

---

## 🛠 Tools Used
- SQL
- Excel (Power Query, Pivot Tables, Charts)
- Google Sheets

---

## 🚀 Business Impact
- Identified root causes of complaint increase
- Enabled faster decision-making through dashboards
- Improved reporting efficiency by ~80%
